## 每 cycle 执行流程

1. **读** `HANDOFF.md` → 拿当前版本 N
2. **读 / 建** `PROGRESS_LOOP.md` → 找已完成项 → 选下一未完成 X.Y
3. **检查停止条件**（见下方），任一命中 → 立即 exit 0
4. **TaskCreate** task: `cycle_<N>: 实施 X.Y <subject>`
5. **改文件**落地实施
6. **起 mock server**（PORT 5180-5199 随机，`CLAUDE_BIN=/tmp/mock-claude.sh`）
7. **验证**：curl / playwright 截图 至少 1 个场景
8. **关 server**
9. **更新** `HANDOFF.md`（在已完成段加 vX.X 子段）
10. **追加** `PROGRESS_LOOP.md`：`cycle_<N> @ <ISO时间> | v<X.X> | <X.Y subject> | substantive | files: <a,b,c>`
11. **标 task completed**
12. **输出 cycle 汇报**（中文 1 段）
13. **exit**（cron 30 分钟后再唤醒）

## 停止条件（任一命中即 exit，不再 cron）

| 信号 | 行为 |
|---|---|
| 项目根存在 `STOP` 文件 | 立即 exit，**调用 CronDelete 取消调度** |
| `HANDOFF.md` 顶部含 "⏸ PAUSED" 字样 | 同上 |
| 任务池全部完成 | 写 `ALL_DONE.md` + CronDelete + 汇报 |
| 单次会话累计 cycle ≥ 20（看 PROGRESS_LOOP.md 行数） | 写 `BUDGET_HIT.md` + CronDelete |
| Claude Max 触发限流（响应含 429 / rate_limit） | 写 `RATE_LIMITED.md` + 暂停 1h（不 Delete） |
| 单 cycle wall clock > 30 min | 跳过当前 X.Y，标 BLOCKED 写 `ERROR_LOG.md`，继续下一项 |

## 备注

- 本 loop 由 CronCreate (in-memory) 调度，**Claude Code 退出即死**。重启 Claude Code 后用户需重新跑 CronCreate
- cron 表达式：`7,37 * * * *`（每小时 :07 / :37，避开 :00/:30 撞点）
- 7 天后 cron 自动过期（CronCreate 内置）
- 用户随时 `touch /Users/hxx/Desktop/00_项目/05_Claude可视化面板/STOP` 立即停止
