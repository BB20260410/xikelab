- JS 文件的 source evidence 已从 regex POC 升级为 Acorn AST 优先：记录 parser、diagnostics、symbols/imports/anchors/references；解析失败、TypeScript/JSX 和其他文本语言继续走 fallback。
- 新增 Codebase Map POC：扫描当前工程文本代码文件，按任务 query 选出 focus files，并生成 evidence、signals 和轻量 import graph。
- 新增 Symbol Graph POC：从 focus files 的 symbols 建立 definitions / references / calls，并把 `/api/...` route anchors 与前端 API 使用关联起来；AST 文件按“同文件定义优先、显式 import 目标次之、唯一全局定义兜底”减少同名 helper 串错。
- Agent 图谱 Dispatch Preview 已支持粘贴受影响文件路径、读取当前 git 变更或构建工程地图，结果会展示 `Code Context`、`Codebase Map`、`Symbol Graph`、`Code Evidence`、`codeScore`、触发理由和路径。
- `metrics.recorded` 会保留 `agentCodeContextSignals` 与 `agentCodeContextEvidence`，让真实运行后的工程上下文分派依据可审计。
- Agent Run 会话化一期已落地：`agent_runs`、`agent_messages`、`agent_tool_results` 三张表进入 SQLite；`MetricsStore.record()` 在 server runtime 中会同步生成 metric 来源的 finished agent run；`POST /api/delegations/:id/autostart` 会同步生成 `delegation_autostart` 来源的 queued agent run；`/api/agent-runs` 支持 run 列表、timeline、message/tool-result 追加、状态迁移和 `format=json|markdown` 导出。
- Agent Run 底座已补上治理关联字段：`deferReason`、`approvalId`、`budgetIncidentId`、`delegationId`、`relatedActivityIds`；run 创建、状态迁移、message 追加、tool result 记录会写入 `agent_run` Activity，后续可从 Activity 反查 run。
- Agent Run 生命周期最小闭环已接入 `RoomAdapter.chat()`：adapter 调用开始创建 running run；预算 hard stop 转 `deferred` / `budget_blocked`；成功转 `succeeded`；失败转 `failed`；abort/cancel 转 `cancelled`。adapter result/error 会携带 `agentRunId`，`MetricsStore.record()` 复用该 run 追加 metric message，避免重复生成 metrics-only run。
- Agent Run 治理链路已打通：budget usage / incident、approval payload、delegation payload、Autopilot job payload、`delegation.autostart` Activity 都会写入同一个 `agentRunId`；Autostart handler 在 `approval_pending`、`budget_blocked`、成功创建/启动目标房时推进同一个 run；run export 会聚合 approval / delegation / autopilot / budget 相关 Activity。
- Codebase Index 二期后端已落地：`CodebaseIndexStore` 提供 rebuild / status / query，本地缓存 Codebase Map；`CodebaseQueryEngine` 负责中文治理问题 token 扩展、源码优先排序、symbol / anchor / snippet / reference 打分；`/api/codebase-index/rebuild`、`/api/codebase-index/status`、`/api/codebase-index/query` 已注册到 server。
- Codebase Index 查询结果保持可解释：返回 `path`、`line`、`score`、`reason`、`anchor`、`parser`、`symbols`、`routes`，不做黑盒 AI 总结；扫描继续跳过 `.git`、`node_modules`、构建产物、`public/vendor`、二进制和超大文件，同时把上限调到能覆盖当前 393KB 的 `public/app.js`，用于回答 Agent 图谱 DOM/handler 问题。
- Codebase Index 5 问 smoke 已验证能命中实现文件：`RoomAdapter` 预算处理、`MetricsStore` / `ActivityLog` 诊断写入、`DelegationAutostart` 链路、`public/app.js` 的 `btnAgentRegistry` handler、`SymbolGraph` route usage 聚合。
- Agent Center 最小 UI 已落地：`🧠 Agent 图谱` 弹窗拆成 `Profiles` / `Dispatch` / `Runs` / `Policies` 四个 tab；默认保留 Dispatch Preview，Runs tab 接入 `/api/agent-runs`、timeline 和 export activity，显示 status、room/session/task、profile、token/cost/latency、diagnostics、messages、tool results，并可跳转审批、委派和 Activity。
- Codebase Center 最小 UI 已落地：顶栏 `⌘` 入口打开本地代码索引面板，显示 index status、Rebuild、Query、results、reason/anchor/snippet/symbols/routes；结果可逐条或批量添加到 Agent Center 的 Dispatch Preview，转成 `affectedFiles` 和最小 `codeContextEvidence`，不做黑盒 AI 总结、不做云索引。
- 权限治理扩展已落地统一判定函数：`evaluatePermission({ actorType, agentRunId, roomId, action, target, cwd, risk })` 返回 `allow/ask/deny + reason + approvalPayload`，并记录 `permission.decision` Activity；带 `agentRunId` 时会追加 Agent Run `decision` message。
- 已接入高风险入口：Claude `Bash/Write/Edit/MultiEdit/Read/LS/Glob/Grep` tool_use、plugin install/delete/reload/exec、MCP create/update/delete/test/list tools/resources/prompts、Room Adapter/Watcher provider-model 配置写入、Watcher provider access 测试、Webhook create/update/test 出站上传、报告 outputPath 写入。
- 权限覆盖范围：危险 shell、文件写/删、外部目录访问、敏感 `.env` 读取、skill/plugin 执行与配置、provider/model config 访问与写入、network upload、auto-accept scope。`deny` 不执行；`ask` 创建 manual approval 并返回 202；私网/loopback upload 和敏感目录直接拒绝。
- 审批后安全重试闭环已补：HTTP/API 高风险入口可从 body/query/header 读取 `approvalId`，只有 action 和 target 与原审批 payload 完全匹配且状态为 `approved` 时才转 `allow`；pending 复用原审批不重复创建；rejected/cancelled/mismatch 直接 `deny`。MCP/Webhook/plugin install 写入前会剥离 `approvalId` 等治理字段，避免污染业务配置。危险终端命令仍不自动重放。
- 旧 SQLite `agent_runs` 表迁移顺序已修复：先补 `delegation_id` / `approval_id` / `source_type` 等列，再创建相关索引，避免旧库访问 `/api/activity`、`/api/agent-runs`、`/api/delegations` 时报 `no such column: delegation_id`。
- 左侧栏入口已放大并分组：`工作台`、`治理`、`系统` 三组，两列大按钮，每个入口有可见名称；窄屏保持单列图标；`btnInspectorToggle` 只更新图标，不再清空“右栏”标签。

## Crow5 学习研究进度与后续方向

当前这条线不是“复刻 Crow5 UI”，而是研究 Crow5 的技术精髓，然后把适合 Xike Lab 的部分吸收为本地、可审计、可测试的能力。按 `docs/crow5-deep-analysis-personal-growth-plan.md` 的框架估算：

- 多 Agent / Skill Registry：约 45% 完成。已落地 profile、dispatch tags、Skill 绑定、诊断、prompt runtime context、profile governance；还缺子 Agent run/session 化、真实并发任务状态、主 Agent 汇总结论视图。
- 本地治理 / 审批 / 审计 / 预算：约 65% 完成。Paperclip 风格的预算、审批、审计、委派、自启动已经可运行；还缺统一 Governance Center、按任务链路追溯、审批/预算/Agent run 的统一 timeline。
- 全工程代码理解：约 25% 完成。已从路径 signals 进化到 Code Evidence、Codebase Map、Acorn AST 优先 references、Symbol Graph；还缺 TypeScript/JSX AST、LSP/Tree-sitter、SQLite FTS/BM25、VectorIndex 融合、增量索引和代码问答引用链。
- 桌面开发体验：约 30% 完成。Xike Lab 仍是 Node/Express + Web UI + Electron packaging；Crow5 的 Tauri + sidecar + SQLite session/message/part 架构还处于研究吸收阶段，不建议马上迁移主项目。
- 模型/provider 生态：约 25% 完成。Xike Lab 有多 adapter，但还没有 Crow5 那种 provider onboarding、模型缓存、用量额度 UI、自定义模型管理和模型选择策略。
- Skill 市场/插件生态：约 35% 完成。已有本地 SkillStore、注入与诊断；还缺分类、安装来源、duplicate/rank、权限策略、usage analytics、插件 SDK 化。
- 产品主路径 Idea -> Plan -> Work -> Verify -> Archive：约 20% 完成。底层能力不少，但用户第一屏还不像 Crow5 那样围绕“把想法做成软件”组织。

总体判断：已经完成 Crow5/Paperclip 精髓中“本地治理 + Agent/Skill 可解释调度”的第一层吸收，距离“真正超越 Crow5”还差核心三块：**可会话化的多 Agent run、真实代码库索引、Idea-to-Archive 产品主路径**。

## 下个阶段优先路线

不要重新泛泛研究，按下面顺序做，每一步都要有测试和 UI/接口证据。

### P0. Agent Run 会话化

目标：把 Crow5 的 `session/message/part/tool-output/session_diff` 思路做成 Xike Lab 自己的 `agent_runs` 抽象。

建议落地：

- 新增 `src/agents/AgentRunStore.js` 或接入现有 metrics/audit store，记录 `runId`、`roomId`、`task`、`chiefProfileId`、`subAgentProfileIds`、`status`、`startedAt/endedAt`。
- 记录每个子 Agent 的 message/tool summary、approval gate、budget preflight、result summary。
- Activity Center 反向链接到 Agent 图谱：从某次 audit event 能跳到对应 profile、skills、evidence、symbolGraph。
- UI 上先做“最近 Agent Runs”小面板，不急着重构全局页面。

验收：用户输入一个任务后，可以看到“谁被派出、为什么派出、用了哪些 Skill、做了什么、是否被审批/预算阻断”。

### P0. Codebase Index 二期

目标：从当前 AST 优先 POC 继续往 Crow5 的 codesearch / 全工程理解推进。

建议落地：

- 给 `JavaScriptAstAnalyzer` 补 TypeScript/JSX 路径：优先考虑 Tree-sitter 或 TypeScript compiler API，不要用更复杂的 regex。
- 新增 SQLite FTS5 或轻量 BM25 索引，和现有 `VectorIndex` 并行，而不是用向量替代符号检索。
- `SymbolGraph` 增加 import/export 名称绑定，处理 re-export、default export、renamed import。
- Codebase Map 增加增量缓存：基于 mtime/size/hash，只重算变化文件。
- 新增 `/api/agent-registry/code-search` 或 `/api/codebase/search`，返回 path、line、reason、symbol/semantic score。

验收：问“RoomAdapter 在哪里接入预算？”必须返回路径、行号、命中原因，并能进入 Agent prompt。

### P0. Idea-to-Archive Flow

目标：把已有治理、委派、Autopilot、Agent 图谱包装成用户能理解的主路径。

建议落地：

- 增加一个“新任务/Idea”入口：用户一句话 -> plan preview -> 建议 Agent -> 预算/审批预估 -> 执行/委派。
- 执行后生成本地归档：任务目标、涉及文件、Agent run、预算/审批、测试结果、截图、后续行动。
- 不要把 Xike Lab 改成 SaaS，不要做企业工单系统；这是本机 AI 开发工作台的任务闭环。

验收：一个任务能从输入、分派、执行、验证、审计、归档完整走通。

### P1. Model / Skill Center

目标：学习 Crow5 的 provider/model/skill 管理体验，但做成 Xike Lab 本地版。

建议落地：

- Provider registry：provider 状态、最近使用、模型列表缓存、健康检查、错误摘要。
- Model picker：按任务 tag 推荐模型，例如 verification 用低成本快模型，architecture 用强模型，privacy/local 用 Ollama。
- Skill Center：分类、绑定关系、冲突/重复提示、每次注入的 usage analytics。

验收：用户不用编辑 JSON，也能知道当前模型/Skill 是否可用，以及为什么被注入。
