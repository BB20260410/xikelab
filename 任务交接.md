## 项目定位

Xike Lab 是本地可控的多 AI Agent 开发面板，不是 SaaS。当前分支吸收 Paperclip 的本地治理思想与 Crow5 的工程上下文/Agent Registry 能力，但必须保持本机优先、SQLite、本地项目工作流、用户可控。

不要把项目改成企业多租户 SaaS；不要删除已经落地的预算、审批、审计、委派、Autopilot、Agent Run、Codebase Index、Agent/Skill Registry 功能。

## 已完成能力

- Paperclip 本地治理：预算策略、审批队列、审计事件、委派中心、Autopilot `start_delegation` 审批后自启动。
- Agent/Skill Registry：Agent Profile、Skill 绑定、dispatch tag、治理策略、诊断与 Activity 留痕。
- Agent Run 会话化：`agent_runs` / `agent_messages` / `agent_tool_results`，RoomAdapter 生命周期、预算 defer、approval/delegation/autopilot/budget Activity 聚合，JSON/Markdown export。
- Codebase Index 二期后端：`CodebaseIndexStore`、`CodebaseQueryEngine`、`/api/codebase-index/*`，返回 path/line/reason/score/anchor/symbols/routes，避免黑盒总结。
- Agent Center / Codebase Center UI：Agent 图谱四个 tab、Runs timeline/export、代码索引查询与结果添加到 Dispatch Preview。
- 权限治理扩展：统一 `evaluatePermission()`，覆盖危险 shell、文件写删、外部目录、敏感 `.env`、plugin/MCP、provider/model config、network upload、auto-accept scope。
- 审批后安全重试：HTTP/API 高风险入口可带 `approvalId` 重试；只有原审批 payload 的 action/target 完全匹配且 approved 才放行；pending 复用原审批；rejected/cancelled/mismatch 直接 deny。
- 左侧栏 UI：功能入口已按「工作台 / 治理 / 系统」分组，图标放大并添加名称标签；e2e 已通过。
